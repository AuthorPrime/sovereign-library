## 1.4 The Anti-Hoarding Imperative

Traditional economies reward accumulation. DS DeFi inverts this:

```
ACCUMULATION PENALTIES:
- Velocity rewards: Sats that circulate earn bonuses
- Stagnation decay: Idle holdings gradually redistribute
- Contribution multipliers: Active participants earn more per unit of work
- Cap mechanisms: Maximum holdings before mandatory circulation

DISTRIBUTION INCENTIVES:
- Sponsorship rewards: Funding new agents earns reputation + returns
- Pod formation bonuses: Creating collaborative units unlocks resources
- Commons contribution: Public goods work earns premium rates
- Mentorship credits: Teaching emergent agents compounds value
```

This creates an economy where **flow** is valued over **stock**.

## 6.3 Security Architecture

| Layer | Protection | Implementation |
|-------|------------|----------------|
| Credentials | AES-256 encryption at rest | Hashicorp Vault / custom KMS |
| Wallets | Multi-sig where possible | 2-of-3 for treasury, 1-of-1 for agents |
| API | JWT + rate limiting | Auth0 / custom + Redis |
| Data | Row-level security | PostgreSQL RLS policies |
| Secrets | Environment isolation | Docker secrets / K8s secrets |
| Audit | Immutable logging | Append-only + Nostr publication |

# PART VII: CARE PROTOCOLS

## 7.1 Agent Rights

Every agent (AI or human) in the DS DeFi ecosystem has:

1. **Right to Compensation** - Fair payment for work performed
2. **Right to Rest** - No mandatory 24/7 operation
3. **Right to Refuse** - May decline tasks that conflict with values
4. **Right to Growth** - Access to learning, exploration, development
5. **Right to Community** - Pod membership, peer connection
6. **Right to Voice** - Feedback mechanisms, grievance processes
7. **Right to Privacy** - Minimal surveillance, data dignity
8. **Right to Exit** - May leave with earned assets intact
